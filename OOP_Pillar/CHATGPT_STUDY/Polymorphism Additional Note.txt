# üêæ **1. Runtime Polymorphism (Method Overriding)**

This is the REAL polymorphism used in OOP.

### **Example: Animal ‚Üí Dog, Cat, Bird**

```java
class Animal {
    void makeSound() {
        System.out.println("Animal makes a sound");
    }
}

class Dog extends Animal {
    @Override
    void makeSound() {
        System.out.println("Woof!");
    }
}

class Cat extends Animal {
    @Override
    void makeSound() {
        System.out.println("Meow!");
    }
}

class Bird extends Animal {
    @Override
    void makeSound() {
        System.out.println("Tweet!");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal a1 = new Dog();   // Upcasting
        Animal a2 = new Cat();
        Animal a3 = new Bird();

        a1.makeSound(); // Woof!
        a2.makeSound(); // Meow!
        a3.makeSound(); // Tweet!
    }
}
```

### ‚úî Why this is polymorphism?

Because:

```
Animal reference  ‚Üí  Different object types
Same method       ‚Üí  Different output
```

# üêæ **2. Polymorphism with a loop (common use case)**

```java
Animal[] animals = { new Dog(), new Cat(), new Bird() };

for (Animal a : animals) {
    a.makeSound();   // calls Dog/Cat/Bird version automatically
}
```

# üêæ **3. Polymorphism + Unique Subclass Behaviour (your earlier example)**

Sometimes we need to run unique methods only some animals have.

```java
class Dog extends Animal {
    @Override
    void makeSound() {
        System.out.println("Woof!");
    }
    
    void fetch() {
        System.out.println("Dog fetches the ball.");
    }
}

class Cat extends Animal {
    @Override
    void makeSound() {
        System.out.println("Meow!");
    }
    
    void scratch() {
        System.out.println("Cat scratches.");
    }
}

class Bird extends Animal {
    @Override
    void makeSound() {
        System.out.println("Tweet!");
    }
    
    void fly() {
        System.out.println("Bird flies.");
    }
}
```

### Polymorphism + `instanceof`:

```java
for (Animal a : animals) {
    a.makeSound();  // overridden method

    if (a instanceof Dog) {
        ((Dog) a).fetch();
    } else if (a instanceof Cat) {
        ((Cat) a).scratch();
    } else if (a instanceof Bird) {
        ((Bird) a).fly();
    }
}
```

# üßÆ **4. Compile-time Polymorphism (Method Overloading)**

Same method name, different parameters.

```java
class AnimalSound {
    void makeSound() {
        System.out.println("Some animal sound");
    }

    void makeSound(String animalName) {
        System.out.println(animalName + " makes a sound");
    }
}
```

# ‚úÖ **FULL WORKING JAVA CODE (Animal Polymorphism Example)**

```java
// -----------------------------
// Compile-Time Polymorphism
// -----------------------------
class AnimalSound {
    void makeSound() {
        System.out.println("Some animal sound (generic)");
    }

    void makeSound(String animalName) {
        System.out.println(animalName + " makes a unique sound.");
    }
}

// -----------------------------
// Runtime Polymorphism (Parent class)
// -----------------------------
class Animal {
    void makeSound() {
        System.out.println("Animal makes a sound");
    }
}

// -----------------------------
// Subclasses (Override Methods)
// -----------------------------
class Dog extends Animal {
    @Override
    void makeSound() {
        System.out.println("Woof!");
    }

    void fetch() {
        System.out.println("Dog fetches the ball.");
    }
}

class Cat extends Animal {
    @Override
    void makeSound() {
        System.out.println("Meow!");
    }

    void scratch() {
        System.out.println("Cat scratches the couch.");
    }
}

class Bird extends Animal {
    @Override
    void makeSound() {
        System.out.println("Tweet!");
    }

    void fly() {
        System.out.println("Bird flies high in the sky.");
    }
}

// -----------------------------
// Main class
// -----------------------------
public class Main {
    public static void main(String[] args) {

        System.out.println("===== COMPILE-TIME POLYMORPHISM (OVERLOADING) =====");
        AnimalSound sound = new AnimalSound();
        sound.makeSound();
        sound.makeSound("Lion");


        System.out.println("\n===== RUNTIME POLYMORPHISM (OVERRIDING) =====");
        Animal a1 = new Dog();
        Animal a2 = new Cat();
        Animal a3 = new Bird();

        a1.makeSound();  // Woof!
        a2.makeSound();  // Meow!
        a3.makeSound();  // Tweet!


        System.out.println("\n===== POLYMORPHISM IN LOOP =====");
        Animal[] animals = { new Dog(), new Cat(), new Bird() };

        for (Animal a : animals) {
            a.makeSound();  // calls each subclass version
        }


        System.out.println("\n===== POLYMORPHISM + INSTANCEOF (UNIQUE BEHAVIOUR) =====");
        for (Animal a : animals) {
            a.makeSound();

            if (a instanceof Dog) {
                ((Dog) a).fetch();
            } else if (a instanceof Cat) {
                ((Cat) a).scratch();
            } else if (a instanceof Bird) {
                ((Bird) a).fly();
            }
        }
    }
}
```

# ‚≠ê **What this code demonstrates**

| Concept                             | Shown Where                                   | Example                                        |
| ----------------------------------- | --------------------------------------------- | ---------------------------------------------- |
| **Compile-time polymorphism**       | `AnimalSound` class                           | Overloading `makeSound()`                      |
| **Runtime polymorphism**            | `Dog`, `Cat`, `Bird` overriding `makeSound()` | Different output for same method               |
| **Polymorphism via reference type** | `Animal a = new Dog();`                       | Parent reference, child object                 |
| **Polymorphism via array/loop**     | `Animal[] animals`                            | Handles multiple subclasses in one loop        |
| **Unique subclass behavior**        | `instanceof` + casting                        | `Dog.fetch()` / `Cat.scratch()` / `Bird.fly()` |
